# Gini AI 프로젝트 개발 보고서

**작성일**: 2025-11-24
**분석 범위**: 전체 코드베이스 및 최근 30개 커밋
**프로젝트 버전**: Alpha/Beta (완성도 약 91%)

---

## 📋 Executive Summary

Gini AI는 PDF 문서를 업로드하면 AI 아바타가 발표하는 영상을 자동 생성하는 Next.js 15 기반 웹 애플리케이션입니다. 현재 91% 완성도로 Alpha/Beta 단계에 있으며, 핵심 기능은 모두 구현되었으나 테스트 인프라 및 에러 복구 로직 강화가 필요한 상태입니다.

### 주요 지표

| 항목 | 수치 |
|------|------|
| 전체 파일 수 | 106개 (TS/TSX) |
| 데이터베이스 테이블 | 11개 |
| API Routes | 16개 |
| Inngest 함수 | 8개 |
| 외부 API 통합 | 4개 (Vertex AI, ElevenLabs, D-ID, Supabase) |
| 최근 30일 커밋 수 | 30개 (버그 수정 67%, 기능 추가 30%) |
| 테스트 커버리지 | 0% (테스트 부재) |

## 🏗️ Architecture Overview

### 5-Layer Architecture

```
┌─────────────────────────────────────────┐
│  1. Presentation Layer (app/)          │
│     - 16 API Routes                    │
│     - 12 Pages (Auth, Dashboard)       │
│     - Server/Client Components         │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  2. Application Layer (lib/inngest/)   │
│     - 8 Background Jobs                │
│     - SceneProcessor Orchestrator      │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  3. Business Logic (lib/services/)     │
│     - Gemini Service (대본 생성)        │
│     - ElevenLabs Service (TTS)         │
│     - D-ID Service (아바타)             │
│     - FFmpeg Service (합성)             │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  4. Data Layer (prisma/)               │
│     - 11 Tables                        │
│     - ReBAC 권한 시스템                 │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  5. External Services                  │
│     - Google Vertex AI                 │
│     - ElevenLabs TTS                   │
│     - D-ID Avatar                      │
│     - Supabase Storage                 │
└─────────────────────────────────────────┘
```

### Video Generation Workflow

```
PDF 업로드 (Supabase Storage)
  ↓
스크립트 생성 (Gemini 2.5 Pro)
  ├─ PDF 분석
  ├─ 씬 분할 (8초 단위)
  └─ 대본 생성 (60자 제한)
  ↓
씬별 순차 처리 (SceneProcessor)
  ├─ TTS 생성 (ElevenLabs)
  ├─ 아바타 생성 (D-ID)
  │   ├─ Talk API 호출
  │   └─ 폴링 (5초 간격, 최대 20회)
  └─ 배경 생성
      ├─ High: Veo 3.1 영상
      ├─ Medium: Nano Banana 이미지 → Veo 업그레이드
      └─ Low: FFmpeg 그라데이션
  ↓
최종 합성 (FFmpeg)
  ├─ 아바타 + 배경 합성
  ├─ 배경음악 믹싱 (40% 볼륨)
  └─ 최종 영상 다운로드
```

## 💾 Database Schema

### 11개 테이블 구조

#### Core Models (7개)
1. **Organization** - 조직 관리 (멀티테넌트)
2. **User** - 사용자 (admin/member)
3. **Project** - 영상 프로젝트 (상태: draft → rendered)
4. **Document** - PDF 업로드 (Supabase Storage)
5. **Scene** - 씬별 대본 및 상태 (TTS/Avatar/Background 독립 상태)
6. **Asset** - 생성된 자산 (audio, avatar_video, background_image/video, final_video)
7. **RenderJob** - D-ID 작업 추적 (Talk ID, 폴링 상태)

#### NextAuth (2개)
8. **Account** - OAuth 계정
9. **Session** - 세션 관리

#### ReBAC (2개)
10. **RelationTuple** - 권한 튜플 (user-namespace-object-relation)
11. **RelationDefinition** - 권한 정의 (owner → editor → viewer 계층)

### ERD 요약

```
Organization (1:N) → User (1:N) → Project (1:1) → Document
                                      ↓ (1:N)
                                    Scene (1:N) → Asset
                                      ↓ (1:N)
                                   RenderJob
```

## 🔌 External API Integration

### 1. Google Vertex AI

#### Gemini 2.5 Pro (대본 생성)
- **용도**: PDF 분석 + 대본 생성
- **입력**: PDF Base64 + 영상 길이 (30/60/180초)
- **출력**: JSON Schema 기반 씬 배열
- **검증**: 60자 제한, 인사말 금지, 괄호 표현 금지
- **비용**: ~$0.50/프로젝트

#### Nano Banana (이미지 생성)
- **용도**: 커스텀 아바타 + 씬 배경 이미지
- **입력**: 텍스트 프롬프트 + aspect ratio (16:9)
- **출력**: 이미지 URL
- **비용**: ~$0.039/이미지

#### Veo 3.1 (영상 생성)
- **용도**: 씬 배경 영상 (image-to-video)
- **입력**: 이미지 URL + 프롬프트 + duration (TTS 길이 기반)
- **출력**: LRO 작업 ID → 폴링 → 영상 URL
- **폴링**: 5초 간격, 최대 20회 (100초 타임아웃)
- **Fallback**: 실패 시 이미지로 대체
- **비용**: ~$1.50/씬 (High priority)

### 2. ElevenLabs TTS

- **용도**: 텍스트 → 음성 변환
- **모델**: eleven_multilingual_v2
- **음성**: Rachel (기본값)
- **설정**: stability 0.5, similarity_boost 0.75
- **출력**: MP3 오디오 + 길이 (FFprobe로 측정)
- **비용**: ~$0.20/씬

### 3. D-ID Avatar

- **용도**: 아바타 립싱크 영상 생성
- **입력**: 아바타 이미지 URL + 오디오 URL
- **API**: Talk API (Mode B: 사전 생성 오디오)
- **폴링**: 5초 간격, 최대 20회
- **상태**: created → processing → done/error
- **비용**: ~$1.50/씬

### 4. Supabase Storage

- **용도**: 파일 저장 (PDF, 오디오, 영상, 이미지)
- **버킷**: gini-ai-assets
- **기능**: 업로드, 다운로드, 서명된 URL 생성
- **최적화**: 대용량 파일 지원 (멀티파트 업로드)
- **비용**: Free tier (1GB 이하)

## ⚙️ Background Jobs (Inngest)

### 8개 함수 구조

1. **sceneProcessor** (오케스트레이터)
   - 씬별 순차 처리 조정
   - TTS → Avatar → Background 순서 보장
   - 커스텀 아바타 대기 로직

2. **avatarDesignGenerator**
   - Nano Banana로 커스텀 아바타 생성
   - Project.avatarDesignSettings 기반

3. **ttsGenerator**
   - ElevenLabs TTS 생성
   - FFprobe로 오디오 길이 측정
   - Asset 생성 + Scene 업데이트

4. **avatarGenerator**
   - D-ID Talk API 호출
   - RenderJob 생성

5. **avatarPolling**
   - D-ID Talk 상태 폴링 (5초 간격, 최대 20회)
   - 완료 시 Asset 생성

6. **backgroundGenerator**
   - Priority 기반 배경 생성 전략
   - High: Veo 영상 생성 요청
   - Medium: Nano 이미지 생성
   - Low: FFmpeg 그라데이션 생성

7. **veoVideoGenerator**
   - Veo 3.1 영상 생성 요청
   - TTS 길이 기반 duration 동적 설정
   - LRO 작업 ID 저장

8. **veoVideoPolling**
   - Veo LRO 폴링 (5초 간격, 최대 20회)
   - 완료 시 Supabase Storage 업로드
   - 실패 시 Nano 이미지로 fallback

### 비활성화된 함수
- **videoCompositor**: 수동 렌더링 방식으로 변경 (render-download API)
- **videoRender**: 즉시 다운로드 방식으로 변경
